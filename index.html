PHP/SQL Implementation

Validation Code

// Validate Patient ID format
if (!preg_match('/^RSPCA-\d{4}-[A-Z0-9]{5}$/', $patientId)) {
    die("❌ Invalid Patient ID format");
}

// Validate condition range
if ($condition < 1 || $condition > 5) {
    die("❌ Condition must be 1-5");
}

// Validate required fields
if (empty($patientId) || empty($condition)) {
    die("❌ All required fields must be filled");
}

SQL Insert Statement

// Embedded SQL in PHP
$sql = "INSERT INTO PATIENT (
    PatientID, 
    SpeciesID, 
    AdmissionDate, 
    ConditionOnArrival, 
    CurrentStatus
) VALUES (
    '$patientId',
    '$speciesId', 
    '$admissionDate', 
    $condition, 
    '$currentStatus'
)";

// Execute query
if ($conn->query($sql) === TRUE) {
    echo "✅ Record inserted successfully";
} else {
    echo "❌ Error: " . $conn->error;
}

PATIENT Table Structure

CREATE TABLE PATIENT ( PatientID VARCHAR(20) PRIMARY KEY, SpeciesID VARCHAR(100) NOT NULL, AdmissionDate DATE NOT NULL, ConditionOnArrival INT CHECK (1-5), CurrentStatus VARCHAR(50) NOT NULL );

Validation Rules Implemented

Patient ID Format

Must match: RSPCA-YYYY-XXXXX

Example: RSPCA-2024-001AB

Condition Range

Must be integer between 1 and 5

1=Critical, 5=Excellent

Date Validation

Admission date cannot be in future

Must be valid date format

Duplicate Prevention

Patient ID must be unique

Check before insertion

Form Submission Flow

  1. User fills form → Client-side validation
  2. Submit to PHP → Server-side validation
  3. Validate against database constraints
  4. If valid → Execute SQL INSERT
  5. Return success/error message to user
